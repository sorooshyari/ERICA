Assignment Profiles

Per-cluster assignment fidelity analysis. For each primary cluster, how much do sample assignments leak to other clusters across B=200 Monte Carlo iterations? Diagonal dominance = stable clusters; off-diagonal spread = boundary ambiguity.

Methodology & References

Algorithm

  1. Run ERICA’s iterative subsampling: B=200 Monte Carlo train/test splits (80/20).
  2. For each iteration, cluster the training set and predict cluster labels for the test set.
  3. Accumulate assignments into the CLAM matrix (Cluster Label Assignment Matrix): CLAM[i,j] = number of iterations where sample i was assigned to cluster j.
  4. Normalize each row by its sum to get assignment proportions.
  5. Assign each sample to its primary cluster (argmax of its CLAM row).
  6. For each primary cluster, plot the distribution of assignments to every other cluster — this is the leakage scatter.
  7. Summarize as a K×K heatmap: M[c,j] = mean % of cluster-c samples assigned to cluster j.

Interpretation

  • Diagonal ≥95%: highly stable cluster — samples almost always return to same cluster.
  • Off-diagonal spikes: systematic leakage to a specific neighbor — boundary ambiguity between those two clusters.
  • Diffuse off-diagonal: sample wanders across many clusters — no stable assignment.
  • Diagonal ≈ 1/K: null model — no cluster structure detected (expected at high σ).

Related work

Parmigiani et al. (2023). “Evaluating the replicability of clustering analyses.” Biostatistics. — Defines the CLAM framework and replicability metrics (CRI/WCRI/TWCRI) that ERICA implements. The assignment profile analysis decomposes the CLAM row structure they propose.

Monti et al. (2003). “Consensus clustering: A resampling-based method for class discovery and visualization of gene expression microarray data.” Machine Learning, 52(1–2), 91–118. — Introduced the consensus matrix (co-association of sample pairs), the precursor to CLAM. Assignment profiles extend this from pairwise co-occurrence to per-sample cluster-level decomposition.

Fred & Jain (2005). “Combining multiple clusterings using evidence accumulation.” IEEE TPAMI, 27(6), 835–850. — Evidence accumulation matrix approach. Like CLAM, counts how often samples co-cluster across ensemble runs.

Strehl & Ghosh (2002). “Cluster ensembles — a knowledge reuse framework for combining multiple partitions.” JMLR, 3, 583–617. — Formalized cluster ensemble theory. The CLAM matrix can be viewed as an evidence accumulation matrix from an ensemble of subsampled partitions.

Implementation

Original analysis: MATLAB clam_matrix_sort_18a.m (K=6, hardcoded). Python port: 12_assignment_profiles.py — generalized to arbitrary K, generates scatter grids + heatmaps + CSV stats for all dataset/method/K combinations. Uses ERICA’s .joblib result files directly.
