## Bugs Found

### BUG 1: Double-append of noise_counts in hdbscan_clustering (PRESENT IN HEAD~3 DIFF)

**File:** `erica/clustering.py`, lines 632 and 642
**Severity:** Medium -- data integrity issue

`noise_counts.append(n_noise)` is called at line 632 (unconditionally, after the HDBSCAN test fit) AND again at line 642 (inside the `len(unique_test) == 0` all-test-noise branch). When all test points are noise, the count is appended twice for that iteration, making `len(noise_counts) > n_iterations`.

**Impact:** The `noise_counts` list in the return dict will have the wrong length. It does not affect CRI/WCRI/TWCRI or ARI/AMI computation, but will confuse any downstream analysis of noise statistics (e.g., the example script uses `np.mean(hdb['noise_counts'])`).

**Fix:** Remove line 642 (`noise_counts.append(n_noise)`), since line 632 already handles it.

### BUG 2: `check_dependencies()` crashes with NameError (PRE-EXISTING)

**File:** `erica/utils.py`, lines 275-276
**Severity:** Medium -- function is broken but not called from tests or core

Line 275 has `#optional = ['plotly', 'matplotlib', 'gradio']` (commented out), but line 276 uses `for dep in optional:`, which will raise `NameError`. This function is exported but untested and unused in the core workflow.

**Fix:** Uncomment line 275.
